导航失败重试机制改进 (P6)：

当前的重试是固定5秒延迟，无限次重试。
建议：
引入最大重试次数。
使用指数退避（Exponential Backoff）的重试延迟。
在多次重试失败后，可以考虑进入一个“安全模式”（例如，停在原地，或者尝试导航到一个非常安全的已知点），并发出更严重的警告。
可以创建一个辅助函数来处理重试调度，以避免 goal_result_callback 过于复杂。
